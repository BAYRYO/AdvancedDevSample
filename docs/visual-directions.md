# Directions visuelles

Utilise le bouton palette (en haut a droite) pour changer de direction visuelle sans modifier le code.

## Direction 1 - Studio

- style: moderne, propre, professionnel
- points forts: lisibilite generale, contraste doux
- ideal pour: documentation technique quotidienne

## Direction 2 - Paper

- style: editorial, lecture longue, ambiance "manuel"
- points forts: confort de lecture textuelle
- ideal pour: specs, guides, architecture narrative

## Direction 3 - Terminal

- style: ops/dev, monospaced, contraste fort
- points forts: code blocks, runbooks, diagnostics
- ideal pour: exploitation, troubleshooting, reference API

## Direction 4 - Ocean

- style: contemporain, lumineux, plus "produit"
- points forts: sections visuelles, navigation agreable
- ideal pour: documentation publique et onboarding

## Comment choisir

1. Ouvre 2-3 pages longues (`architecture`, `api`, `quality`).
2. Teste chaque direction 1 minute.
3. Evalue: lisibilite, fatigue visuelle, confort des blocs code, contraste des diagrammes.
4. Garde la direction la plus stable sur desktop + mobile.
