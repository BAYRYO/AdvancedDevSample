# Security Policy

## Supported Versions

Security fixes are applied on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Use GitHub Security Advisories:

- Go to `Security` tab in the repository.
- Create a private vulnerability report.
- Provide clear reproduction steps, affected components, and impact.

## Disclosure Process

1. We acknowledge receipt as soon as possible.
2. We assess impact and remediation scope.
3. We prepare and validate a fix.
4. We coordinate disclosure and release notes.
